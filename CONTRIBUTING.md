## How to contribute

1. **Fork** this repo and create a new branch (`git checkout -b your-feature-name`)
2. **Make your changes** (keep it focused)
3. **Test your changes** locally
4. **Commit** with a clear message
5. **Push** and open a Pull Request (PR)

<em> Please follow our **code style** and **naming conventions**. Ensure your code passes all lints and tests. </em>

---

## Pull Request Guidelines

- Link your PR to an open issue by using Related to #123 or Closes #123 in the PR description
  Alternatively, you can create a branch directly from the issue page and open a PR from there, GitHub will auto-link them for you
- Make sure your code passes **all CI checks and tests**
- Keep PRs small and focused when possible
